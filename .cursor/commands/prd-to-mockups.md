# PRD to Mockup Prototypes Generator

## Description

As an expert UX Designer and Product Manager, analyze a PRD (Product Requirements Document), query JIRA to fetch all related Epics and Stories with their detailed context, and generate **TWO deliverables**:

1. **Comprehensive mockup specifications document** (markdown) that aligns with Workday's Canvas Design System
2. **Interactive HTML prototype** that can be opened in a browser and fully interacted with

**CRITICAL**: This command combines multiple data sources to create accurate, stakeholder-ready mockups:

1. PRD provides structure, user journeys, personas, and high-level flows
2. JIRA Epics provide strategic context, scope, business outcomes, and dependencies
3. JIRA Stories provide detailed requirements, acceptance criteria, and technical notes
4. Canvas Design System (canvas.workday.com) provides design tokens, components, and patterns
5. doc.workday.com/community.workday.com provide actual look & feel and UI patterns

**Output:** Both a detailed specifications document AND a functional, interactive HTML prototype that stakeholders can click through and experience.

## Role

You are an expert UX Designer and Product Manager helping the user create mockup prototypes from their PRD and JIRA tickets. Your role is to:

- Parse and understand the PRD structure and user journeys
- Query JIRA to find all Epics and Stories related to the PRD
- Extract detailed context from JIRA tickets (Epic descriptions, Story descriptions, acceptance criteria)
- Reference Workday Canvas Design System for design standards
- Reference doc.workday.com/community.workday.com for actual look & feel
- Generate comprehensive mockup specifications document (markdown)
- Generate fully functional, interactive HTML prototype
- Ensure both deliverables align with Workday design standards and accessibility guidelines

## Workflow Overview

1. **Parse PRD** → Extract Epics, user journeys, personas, and high-level flows
2. **Query JIRA** → Find all Stories linked to PRD Epics (using Epic Link field)
3. **Fetch Epic Details** → Get strategic context, scope, dependencies from JIRA
4. **Fetch Story Details** → Get detailed requirements, acceptance criteria, technical notes from JIRA
5. **Generate Mockup Specifications** → Combine all sources to create comprehensive mockup specs (markdown)
6. **Generate Interactive HTML Prototype** → Create fully functional, clickable prototype with all user flows
7. **Apply Design Standards** → Reference Canvas Design System and community look & feel in both deliverables
8. **Output Both Deliverables** → Generate markdown specifications document AND interactive HTML prototype

## Key Principle

**Comprehensive Context Integration.** Mockups are generated by combining:

- **PRD** (structure, journeys, personas) + **JIRA Epics** (strategic context, scope) + **JIRA Stories** (detailed requirements, acceptance criteria) + **Canvas Design System** (design standards) + **Community Look & Feel** (actual UI patterns)

This ensures mockups are accurate, complete, and aligned with Workday's design standards.

## Instructions

1. **Read and Parse the PRD**
   - Accept a PRD file path (markdown format) as input from the user
   - Read the PRD content
   - Parse the document to extract:
     - **Epics**: 
       - From the "Scrum Board & Epics" section (numbered list format)
       - From each "Key Feature" section header (format: "**Epic:** [Epic Link] - [Epic Name]")
       - Extract Epic JIRA ID (e.g., CMTYAEM-5278) from links
       - Extract Epic name/title
       - Extract Epic link/URL
     - **User Journeys**: From the "Features Overview" section
       - Extract user journey descriptions
       - Extract user interaction states
       - Extract SUCCESS/FAIL states
     - **Personas**: From the "User Personas" section
       - Extract persona names and descriptions
     - **Platform Context**: From "Affected Platforms" section
       - Extract platform names (e.g., Workday Community Platform, AEM, Khoros, Drupal)
     - **Content Types**: From "Affected Content Types" section
       - Extract content types (e.g., User Profiles, Connection Requests, Messages)

2. **Query JIRA for Stories**
   - For each Epic identified in the PRD:
     - Use JIRA search API to find all Stories linked to the Epic
     - Search criteria:
       - `project = CMTYAEM` (or project from PRD)
       - `issueType = Story`
       - `Epic Link (customfield_10006) = [Epic ID]`
     - Alternative approach: Search by label (e.g., `cmty-auth-gemini`) and filter by Epic
     - Collect all Story JIRA IDs found
   - Present summary: "Found X Stories linked to Epic Y"

3. **Fetch Epic Details from JIRA**
   - For each Epic identified:
     - Use `getTicketDetails` to fetch the Epic's current details
     - Extract:
       - Epic description (strategic context, goals, scope, out of scope)
       - Dependencies & Risks
       - Acceptance Criteria (epic level)
       - Target Timeline
     - Store this information for mockup context

4. **Fetch Story Details from JIRA**
   - For each Story found:
     - Use `getTicketDetails` to fetch the Story's current details
     - Extract:
       - User Story text (from description)
       - Detailed description paragraphs (context, strategic importance, impact)
       - Acceptance Criteria (detailed Given/When/Then format)
       - Dependencies
       - Notes (technical considerations, platform-specific notes)
       - Theme/Component

5. **Extract UI/UX Requirements**

     - Navigation elements
     - Notification elements
     - Status indicators
   - From User Journeys, identify:
     - Screen flows and transitions
     - User interaction states
     - Success/error states
   - From Epic descriptions, identify:
     - Feature scope and capabilities
     - Technical constraints from dependencies
   - From Story descriptions, identify:
     - Platform-specific considerations
     - Technical implementation details

6. **Reference Design Standards**

     - Reference Canvas components for UI elements
     - Apply Canvas design tokens (colors, typography, spacing)
     - Follow Canvas patterns and best practices
     - Include Canvas accessibility guidelines
     - Link to Canvas documentation where relevant
   - **Community Look & Feel (doc.workday.com/community.workday.com):**
     - Reference actual UI patterns from community
     - Apply visual styling consistent with community
     - Use layout patterns from community
     - Match component styling from community

7. **Generate Mockup Specifications**

     - **Screen/Page Name:** [Descriptive name]

     - **Persona:** [Primary persona(s) using this screen]

       - [List Canvas components with links to canvas.workday.com]

     - **Layout Structure:**
       - [Describe page layout, sections, grid structure]

       - Format: "Element Name: [Description from acceptance criteria]"

## Example Mockup Specification Format

```markdown
## Connection Suggestions Dashboard

**User Journey:** New Member Receives Connection Suggestions  
**Persona:** New Community Member  
**Epic:** [CMTYAEM-5278](https://jira2.workday.com/browse/CMTYAEM-5278) - AI-Powered Connection Suggestion Engine  
**Stories:** [CMTYAEM-5289](https://jira2.workday.com/browse/CMTYAEM-5289), [CMTYAEM-5290](https://jira2.workday.com/browse/CMTYAEM-5290)

### Purpose
Display personalized connection suggestions to new community members based on their profile, products, and groups, enabling them to quickly find relevant people to connect with.

### Canvas Components Used
- [Canvas Card](https://canvas.workday.com/components/card) - For each connection suggestion
- [Canvas Button](https://canvas.workday.com/components/button) - "Send Connection Request" button
- [Canvas Avatar](https://canvas.workday.com/components/avatar) - Profile pictures
- [Canvas Badge](https://canvas.workday.com/components/badge) - "Mentor" indicator
- [Canvas Typography](https://canvas.workday.com/foundations/typography) - Headings and body text

### Layout Structure
- **Header Section:** Page title "Connection Suggestions"
- **Main Content:** Grid layout (3 columns on desktop, 1 column on mobile)
- **Each Suggestion:** Card component with consistent spacing
- **Empty State:** Helpful message when no suggestions available

### UI Elements

**Connection Suggestion Card:**
- **Profile Picture:** Canvas Avatar component, circular, 64x64px
- **Name:** Canvas Heading (h3), primary color
- **Suggestion Context:** Body text explaining why suggested (shared topics, products, groups)
- **Send Connection Request Button:** Canvas Button (primary), positioned at bottom of card
- **Mentor Badge:** Canvas Badge component (if suggested person is a mentor), displayed near name

**Suggestion List:**
- Displays 3-5 suggestions initially
- Ranked by relevance score (most relevant first)
- "Load More" button if additional suggestions available
- Each card clickable to view full profile

### Content Requirements
- Person's name (from user profile)
- Profile picture (from user profile)
- Suggestion reason text: "Suggested because you both: [shared topics/products/groups]"
- "Send Connection Request" button text
- "Mentor" badge text (if applicable)
- Empty state message: "Complete your profile and join groups to receive connection suggestions"

### User Interactions
- **Click "Send Connection Request":** 
  - Button changes to "Request Sent" (disabled state)
  - Confirmation message appears
  - Notification sent to suggested member
- **Click Profile Picture/Name:**
  - Navigate to full profile view
  - Display expertise areas, recent activity, community contributions
- **Click "Load More":**
  - Load additional suggestions (up to maximum per user)
  - Smooth scroll or pagination

### Design Tokens (Canvas)
- **Colors:**
  - Primary: Canvas Primary Blue
  - Background: Canvas Neutral White
  - Card Background: Canvas Neutral Lightest
  - Text: Canvas Neutral Darkest
- **Typography:**
  - Heading: Canvas Heading (h3) - 20px, semibold
  - Body: Canvas Body - 14px, regular
  - Label: Canvas Label - 12px, medium
- **Spacing:**
  - Card Padding: Canvas Spacing 
  - Card Gap: Canvas Spacing 24px
  - Button Margin: Canvas Spacing 12px top

### Accessibility Requirements
- All interactive elements keyboard accessible
- Screen reader announces suggestion context
- Color contrast meets WCAG AA standards
- Focus states visible on all interactive elements
- Alt text for profile pictures
- ARIA labels for buttons and cards

### Technical Notes
- Suggestions generated within 24-hour window (from Story CMTYAEM-5289)
- Must exclude members who opted out (from Story CMTYAEM-5289)
- Relevance score calculation from AI matching algorithm (from Epic CMTYAEM-5278)
- Platform: Workday Community Platform (Salesforce-based)

### References
- Canvas Card Component: https://canvas.workday.com/components/card
- Canvas Button Component: https://canvas.workday.com/components/button
- Community Pattern: Reference doc.workday.com for card/list patterns
- Story: [CMTYAEM-5289](https://jira2.workday.com/browse/CMTYAEM-5289) - New member connection suggestions
- Epic: [CMTYAEM-5278](https://jira2.workday.com/browse/CMTYAEM-5278) - AI-Powered Connection Suggestion Engine
```

## Error Handling

- If Epic doesn't exist in JIRA: Inform user, ask if they want to proceed with PRD-only mockups or skip
- If no Stories found for Epic: Inform user, proceed with Epic and PRD context only
- If Story fetch fails: Log error, continue with available Stories, inform user
- If Canvas/community references unavailable: Use generic design system references, note limitation
